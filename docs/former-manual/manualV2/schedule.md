# 排班教学

## 页面介绍

排班编辑界面分为四个功能区域：

1.  条件区：设置触发条件
2.  副表任务区：配置执行任务
3.  设施区：显示工作站布局
4.  设置区：干员管理

操作指南：

+   主表的条件及任务区需通过"新建副表"激活
+   拖拽调整设施区位置需与实际游戏布局一致
+   输入干员名称时可使用首字母缩写快速匹配

## 主力与替换

**主班（高效组）**：核心高效干员，优先安排工作

**替班**：替补干员，在主班休息时自动轮换

轮换规则：

+   Mower选择列表中靠前且未被占用的干员接替工作
+   一名干员可担任多名主力的替换
+   主力选择原则：维持长工作时间高效率状态

验证规则：

+   高效组不可使用龙舌兰、但书（仅限替换组）
+   同一替换组不可同时安排龙舌兰和但书
+   替换组不可包含高效组干员
+   确保替换组数量充足

## 班组管理

填写相同组名可将干员绑定为班组：

+   同组干员一起上下班
+   不同工作站干员也可绑定
+   宿管不可绑组

小组：无中枢的班组

大组：包含中枢的班组

## 宿舍管理

配置要求：

+   每个宿舍安排一名单回宿管和一名群回宿管
+   下层宿舍回复能力不超过上层宿舍
+   单回宿管数值要求：琴柳(0.7)≥蜜莓(0.7)≥车尔尼(0.65)≥波登可(0.65)
+   群回宿管数值要求：流明(0.25/0.4)≥杜林(0.25)≥桃金娘(0.15)≥波登可(0.15)

## 低优先级设置

仅作用于主班干员，配置后：

+   不会主动前往高优位休息
+   高优位充足时可使用
+   高优干员需入驻时会被移出

推荐适用情况：

+   非绑组的会客室/办公室等低收益站干员
+   回复能力显著高于同组成员的干员
+   主副班效率相近的干员

## 心情回复与消耗

宿舍基础回复公式：

```
1.5 + 0.1 × 宿舍等级 + 0.0004 × 氛围值
```

各等级宿舍回复能力：

| 宿舍等级 | 心情回复 |
| --- | --- |
| 1 | 2 |
| 2 | 2.5 |
| 3 | 3 |
| 4 | 3.5 |
| 5 | 4 |

工作站基础消耗：

+   贸易站/制造站：无减免时为1
+   入驻2个干员：0.05减免
+   入驻3个干员：0.1减免

中枢效果：每入驻1个干员全局-0.05消耗（上限0.25）

## 工休比计算

工作时间占比公式：

```
工休比 = 回复效率 / (回复效率 + 消耗效率)
```

肥鸭充能情况：

```
工休比 = (回复效率 + 肥鸭提供效率) / (回复效率 + 消耗效率)
```

应用场景：

+   计算布局收益
+   评估大组轮休可行性
+   确定替换组能替多少人
+   计算肥鸭充能收益
